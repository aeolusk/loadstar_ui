# LOADSTAR Explorer - 구현 기능 목록

## 프로젝트 개요

- **목적**: LOADSTAR 메타데이터 시스템의 탐색/관리용 웹 UI
- **아키텍처**: Spring Boot (로컬 백엔드) + React (CSR 프론트엔드)
- **배포 형태**: 로컬 실행 (`java -jar loadstar-explorer.jar` → localhost 접근)
- **협업 방식**: 각 개발자가 로컬에서 실행, Git으로 데이터 공유

---

## 기능 목록

### F01. 대시보드 (홈)

프로젝트 진입 화면. 현재 상태를 한눈에 파악한다.

- 전체 요소 수 표시 (Map / WayPoint / BlackBox)
- 상태별 분포 (S_IDL / S_PRG / S_STB / S_ERR / S_REV)
- ACTIVE 작업 요약 (현재 진행 중인 WayPoint + 관련 TODO)
- 건강 상태 경고
  - SYNCED_AT 30일 초과 BlackBox 수
  - 미처리 드리프트 항목 수
  - 미결 OPEN_QUESTIONS 수
  - checkpoint_needed.flag 존재 여부

---

### F02. Map 표시 (WayPoint 흐름 표시)

Map별로 독립된 화면을 생성하여 하위 WayPoint의 흐름을 시각화한다.

- Map 목록에서 선택 → 해당 Map 전용 화면 진입
- Map 하위 요소(Map, WayPoint) 트리/그래프 표시
- WayPoint 간 CONNECTIONS (PARENT, CHILDREN, REFERENCE) 관계 시각화
- 각 WayPoint 노드에 STATUS, 진행률(TECH_SPEC 체크박스) 요약 표시
- 노드 클릭 → WayPoint 상세 화면으로 이동

---

### F03. WayPoint 추가/삭제/편집

WayPoint의 전체 생명주기를 관리한다.

- **조회**: IDENTITY, CONNECTIONS, TODO/TECH_SPEC, ISSUE, COMMENT 전체 표시
- **추가**: 부모 요소 선택 → ID 입력 → `loadstar create W` 호출
  - BlackBox 자동 생성 확인
- **편집**: SUMMARY, TECH_SPEC 체크박스, ISSUE, OPEN_QUESTIONS, COMMENT 인라인 편집
  - STATUS 변경 (S_IDL → S_PRG → S_STB 등)
  - CONNECTIONS 편집 (CHILDREN, REFERENCE 추가/제거)
- **삭제**: 확인 다이얼로그 → `loadstar delete` 호출
  - 하위 요소 존재 시 경고

---

### F04. BlackBox 화면 (편집)

코드 수정 전 진입점인 BlackBox를 조회하고 편집한다.

- **조회**: DESCRIPTION, CODE_MAP, TODO(WP_REF), ISSUE, COMMENT 표시
- **CODE_MAP 뷰어**
  - 계획 상태: 파일/함수 단위 목록
  - 실측 상태: 파일:시작줄-끝줄 형식
  - SYNCED_AT 30일 초과 시 드리프트 경고 표시
- **CODE_MAP 편집**: 계획 → 실측 전환, 라인번호 갱신
- **TODO 편집**: WP_REF 연결 상태 확인, 체크 상태 WayPoint TECH_SPEC과 동기화
- **ISSUE 편집**: 제약/미결 문제 추가/수정
- **STATUS 표시**: S_IDL / S_PRG / S_STB / S_REV
- **로그 뷰어**: 해당 BlackBox에 누적된 loadstar log 항목 표시

---

### F05. TODO 목록 표시 (CLI 연동)

`loadstar todo` CLI와 연동하여 TODO를 관리한다.

- **목록 조회**: `loadstar todo list` 결과 표시
  - 상태별 필터: PENDING / ACTIVE / BLOCKED
  - Depends_On 관계에 따른 BLOCKED 자동 표시
- **TODO 추가**: Address(W://) 선택 + Summary 입력 + Depends_On 지정
  - `loadstar todo add` 호출
- **상태 변경**: PENDING ↔ ACTIVE 전환
  - `loadstar todo update` 호출
- **완료 처리**: `loadstar todo done` 호출 → TODO_HISTORY로 이관
- **삭제**: `loadstar todo delete` 호출
- **의존성 표시**: Depends_On 관계를 시각적으로 연결

---

### F06. TODO History 표시

완료/변경/삭제된 TODO의 이력을 조회한다.

- **전체 이력 조회**: `loadstar todo history` 결과 표시
- **Address별 필터링**: 특정 WayPoint의 TODO 이력만 조회
- **Action별 필터링**: DONE / UPDATED / DELETED
- **타임라인 표시**: 시간순 이력 시각화
- **테이블 컬럼**: Address, Time, Summary, Action, At, Depends_On

---

### F07. 모니터링/드리프트 현황

MONITOR_SPEC 기반의 드리프트 감지 및 조정 지원 화면.

- **스냅샷 관리**
  - 현재 스냅샷 생성: `loadstar monitor snapshot` 호출
  - 스냅샷 목록 표시 (MONITOR/SNAPSHOT/ 하위 파일)
- **Diff 비교**
  - 두 스냅샷 간 변경 파일 목록: `loadstar monitor diff` 호출
  - DIFF_LOG 내용 표시
- **드리프트 현황**
  - 미처리 변경 항목 목록
  - checkpoint_needed.flag 존재 시 경고 배너
- **SYNCED_AT 일람**
  - 전체 요소의 SYNCED_AT 날짜 목록
  - 30일 초과 항목 강조 표시
- **조정 세션 가이드**
  - 드리프트 항목 우선순위별 정리
  - 조정 진행 체크리스트

---

### F08. GIT 이력 조회

LOADSTAR checkpoint 및 Git 커밋 이력을 조회한다.

- **최근 커밋 목록**: Git log 요약 (해시, 메시지, 일시, 작성자)
- **커밋 상세**: 변경된 파일 목록, diff 표시
- **LOADSTAR 요소 태그**: 커밋 메시지에서 변경된 요소 주소 추출/표시
- **AUTO-CHECKPOINT 구분**: 자동/수동 커밋 필터링

---

### F09. CLI 기능 수행

LOADSTAR CLI 명령을 UI에서 직접 실행한다.

- **명령 실행 패널**: CLI 명령 입력 → 실행 → 결과 표시
- **지원 명령**:
  - `loadstar checkpoint -m "..."` — checkpoint 실행
  - `loadstar create [TYPE] [ID] --parent [ADDRESS]` — 요소 생성
  - `loadstar delete [ADDRESS]` — 요소 삭제
  - `loadstar show [ADDRESS]` — 요소 조회
  - `loadstar log [ADDRESS] [KIND] "[내용]"` — 로그 추가
  - `loadstar todo [add|update|done|delete|list|history]` — TODO 관리
  - `loadstar monitor [snapshot|diff|status|clean]` — 모니터링
  - `loadstar git [set|status|unset]` — Git 연결 관리
- **실행 결과**: stdout/stderr 표시
- **실행 이력**: 최근 실행한 명령 목록

---

### F10. 로그 검색 및 조회

BlackBox에 누적된 loadstar log 항목을 검색/조회한다.

- **전체 로그 조회**: `loadstar findlog` 결과 표시
- **필터링**
  - Address별: 특정 요소의 로그만 조회
  - KIND별: NOTE / DECISION / ISSUE / RESOLVED / PROGRESS / MODIFIED
- **페이징**: offset + limit 기반 페이지 이동
- **시간순 정렬**: 최신순 기본, 역순 전환
- **검색**: 로그 내용 텍스트 검색

---

### F11. 검색

프로젝트 내 요소를 빠르게 찾는다.

- **주소 검색**: W://, M://, B:// 주소로 직접 이동
- **키워드 검색**: SUMMARY, TECH_SPEC, CODE_MAP 내용 전문 검색
- **상태 필터**: STATUS 코드별 요소 필터링
- **검색 결과**: 요소 유형, 주소, SUMMARY 요약 목록 → 클릭 시 상세 화면 이동
