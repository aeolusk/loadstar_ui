# LOADSTAR Explorer - 주요 업무 흐름

## 서버 (Spring Boot)

서버는 `.loadstar/` 파일 시스템과 CLI를 브라우저에 연결하는 브릿지 역할이다.

### 최상위 WayPoint 구성

```
M://root/backend
├─ W: project_config       ← 프로젝트 초기화 및 설정 (.loadstar 경로 지정, 연결 확인)
├─ W: element_service      ← 요소 파싱/조회/편집 (Map, WayPoint, BlackBox 파일 I/O)
├─ W: todo_service         ← TODO 관리 (CLI 연동: add, update, done, delete, list, history)
├─ W: git_service          ← Git 연동 (이력 조회, checkpoint 실행)
├─ W: monitor_service      ← 모니터링 (스냅샷, diff, 드리프트 현황, SYNCED_AT)
├─ W: log_service          ← 로그 관리 (CLI 연동: log, findlog)
└─ W: cli_service          ← CLI 범용 실행 (명령 입력 → 실행 → 결과 반환)
```

### 업무 흐름

#### 1. 프로젝트 연결

```
클라이언트 → [POST /api/project/open] → 서버
                                         │
                                         ├─ .loadstar/ 존재 확인
                                         ├─ MAP/root.md 파싱 → 프로젝트 정보 반환
                                         └─ Git 상태 확인 (연결 여부, 브랜치, 최근 커밋)
```

#### 2. 요소 트리 조회

```
클라이언트 → [GET /api/elements/tree] → 서버
                                         │
                                         ├─ MAP/ 전체 파일 스캔
                                         ├─ WAYPOINT/ 전체 파일 스캔
                                         ├─ BLACKBOX/ 전체 파일 스캔
                                         ├─ 각 파일의 ADDRESS, STATUS, CONNECTIONS 파싱
                                         └─ 트리 구조 JSON 반환
```

#### 3. 요소 상세 조회/편집

```
클라이언트 → [GET /api/elements/{address}] → 서버
                                              │
                                              └─ 주소 → 파일 경로 변환 → 파싱 → JSON 반환

클라이언트 → [PUT /api/elements/{address}] → 서버
                                              │
                                              ├─ 요청 데이터로 md 파일 갱신
                                              └─ SYNCED_AT 업데이트
```

#### 4. TODO 관리 (CLI 위임)

```
클라이언트 → [GET /api/todo/list] → 서버 → loadstar todo list → 결과 파싱 → JSON 반환
클라이언트 → [POST /api/todo] → 서버 → loadstar todo add [ADDRESS] "[SUMMARY]" → 결과 반환
클라이언트 → [PUT /api/todo/{address}] → 서버 → loadstar todo update [ADDRESS] [STATUS] → 결과 반환
클라이언트 → [DELETE /api/todo/{address}] → 서버 → loadstar todo done [ADDRESS] → 결과 반환
클라이언트 → [GET /api/todo/history] → 서버 → loadstar todo history → 결과 파싱 → JSON 반환
```

#### 5. Git / Checkpoint

```
클라이언트 → [GET /api/git/log] → 서버 → git log 실행 → JSON 반환
클라이언트 → [POST /api/git/checkpoint] → 서버 → loadstar checkpoint -m "..." → 결과 반환
클라이언트 → [GET /api/git/status] → 서버 → loadstar git status → JSON 반환
```

#### 6. 모니터링

```
클라이언트 → [POST /api/monitor/snapshot] → 서버 → loadstar monitor snapshot → 결과 반환
클라이언트 → [GET /api/monitor/diff] → 서버 → loadstar monitor diff → 결과 파싱 → JSON 반환
클라이언트 → [GET /api/monitor/synced-at] → 서버 → 전체 요소 SYNCED_AT 스캔 → JSON 반환
```

#### 7. 로그

```
클라이언트 → [POST /api/log] → 서버 → loadstar log [ADDRESS] [KIND] "[내용]" → 결과 반환
클라이언트 → [GET /api/log/search] → 서버 → loadstar findlog [OFFSET] [LIMIT] --address --kind → JSON 반환
```

#### 8. CLI 범용 실행

```
클라이언트 → [POST /api/cli/exec] → 서버
                                      │
                                      ├─ 명령 문자열 검증 (loadstar 명령만 허용)
                                      ├─ ProcessBuilder로 실행
                                      └─ stdout/stderr 반환
```

## 클라이언트 (React)

클라이언트는 서버 API를 호출하여 화면을 구성하고 사용자 인터랙션을 처리한다.

### 최상위 WayPoint 구성

```
M://root/frontend
├─ W: app_shell            ← 앱 골격 (Menu Bar, Toolbar, Element Tree, Editor Tabs, Status Bar)
├─ W: map_view             ← Map 화면 (React Flow 기반 WayPoint 체인 흐름도)
├─ W: waypoint_editor      ← WayPoint 상세/편집 화면
├─ W: blackbox_editor      ← BlackBox 상세/편집 화면
├─ W: dashboard            ← 대시보드 (프로젝트 개요, 건강 상태)
├─ W: todo_view            ← TODO 목록 + History (탭 전환)
├─ W: monitor_view         ← 모니터링/드리프트 현황
├─ W: git_view             ← GIT 이력 조회
├─ W: log_view             ← 로그 검색/조회
├─ W: cli_console          ← CLI 실행 콘솔
└─ W: search               ← 검색 (Command Palette)
```

### 업무 흐름

#### 1. 앱 초기화

```
앱 로드 → [GET /api/project/open] → 프로젝트 정보 수신
       → [GET /api/elements/tree] → Element Tree 구성
       → [GET /api/todo/list] → 알림 뱃지 업데이트
       → Status Bar에 프로젝트 경로, checkpoint 상태 표시
```

#### 2. Element Tree 탐색

```
트리 렌더링 (Map/WayPoint/BlackBox 계층)
  │
  ├─ 싱글클릭 → 트리 노드 선택 (하이라이트)
  └─ 더블클릭 → Editor Tabs에 해당 요소 탭 열기
                 │
                 ├─ Map → [GET /api/elements/{address}] → Map View 탭 (React Flow 흐름도)
                 ├─ WayPoint → [GET /api/elements/{address}] → WayPoint Editor 탭
                 └─ BlackBox → [GET /api/elements/{address}] → BlackBox Editor 탭
```

#### 3. Map View (React Flow)

```
Map 데이터 수신 → React Flow 노드 배치
  │
  ├─ WayPoint: 사각형 박스, 좌→우 체인 연결
  │    ├─ STATUS 색상 표시
  │    └─ TECH_SPEC 진행률 표시
  │
  ├─ 체인 아래: BlackBox, 참조 WayPoint 연결
  │
  └─ 노드 클릭 → 해당 요소 탭 열기
```

#### 4. WayPoint/BlackBox 편집

```
편집 화면 표시 → 인라인 편집 (SUMMARY, TECH_SPEC, ISSUE 등)
  │
  └─ 저장 → [PUT /api/elements/{address}] → 서버에 반영
          → Element Tree STATUS 갱신
```

#### 5. TODO 관리

```
TODO 탭 열기 → [GET /api/todo/list] → 목록 렌더링
  │
  ├─ 추가 → 폼 입력 → [POST /api/todo] → 목록 갱신
  ├─ 상태 변경 → [PUT /api/todo/{address}] → 목록 갱신
  ├─ 완료 → [DELETE /api/todo/{address}] → 목록 갱신
  └─ History 탭 전환 → [GET /api/todo/history] → 이력 렌더링
```

#### 6. Toolbar 버튼 → 기능 탭

```
Toolbar 버튼 클릭 → Editor Tabs에 기능 탭 열기
  │
  ├─ [대시보드] → 프로젝트 개요 API 호출 → 카드/차트 렌더링
  ├─ [TODO] → TODO 목록 탭
  ├─ [History] → TODO History 탭
  ├─ [모니터링] → 드리프트 현황 API 호출 → SYNCED_AT 테이블 렌더링
  ├─ [GIT] → Git log API 호출 → 커밋 이력 렌더링
  ├─ [로그] → findlog API 호출 → 필터/페이징 렌더링
  └─ [CLI] → 명령 입력 폼 → [POST /api/cli/exec] → 결과 표시
```
